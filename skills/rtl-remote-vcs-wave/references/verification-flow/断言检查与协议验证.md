# 断言检查与协议验证

断言应作为企业级验证平台的标配能力，而不是可选项。

## 为什么要加断言

断言适合发现：
- 协议时序错误
- reset 期间非法行为
- 握手不匹配
- FIFO 溢出/下溢前兆
- 状态机非法迁移

## 推荐放置位置

- interface 中放协议断言
- tb_top 或 checker 中放系统级断言
- 特殊属性可放独立 assertion 文件

## 渐进式接入

### 第 1 批必须有
- reset 后输出稳定性
- valid/ready 基本握手关系
- 非法状态保护

### 第 2 批建议补
- 时序窗口约束
- FIFO 深度边界
- 仲裁公平性基础属性

### 第 3 批进阶
- 复杂跨事务关联属性
- 与 RAL 配置联动的属性
