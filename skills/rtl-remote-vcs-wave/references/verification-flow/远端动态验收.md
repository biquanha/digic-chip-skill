# 远端动态验收

本文定义新模块 UVM 平台在远端执行 smoke 验收时的标准动作。

## 目标

对一个已生成或已接入的 UVM 工程，至少完成以下闭环：
- 远端编译
- 远端仿真
- 生成 FSDB
- 提取日志摘要
- 判断是否通过最小验收

## 推荐脚本

```bash
bash skills/rtl-remote-vcs-wave/scripts/run_remote_uvm_smoke.sh   --remote eda01   --workdir /nfs/home/USER/project/demo_fifo   --testname smoke_test
```

## 默认约定

脚本默认假设：
- 工程根目录下存在 `sim/Makefile`
- `make comp` 用于编译
- `make sim` 用于仿真
- 仿真输出 `sim.log`
- 波形文件为 `tb.fsdb`

如果用户工程不同，agent 应先适配 Makefile 约定，再执行脚本。

## 通过判据

最小 smoke 验收至少检查：
- `simv` 生成成功
- `tb.fsdb` 成功生成
- `UVM_ERROR == 0`
- `UVM_FATAL == 0`
- scoreboard 的 `error_count == 0`

## 产物

脚本执行后，建议检查：
- `sim/vcs_compile.log`
- `sim/sim.log`
- `sim/tb.fsdb`
- `sim/uvm_smoke_summary.txt`
- `sim/fsdb_report.txt`（如果 `fsdbreport` 生成成功）
