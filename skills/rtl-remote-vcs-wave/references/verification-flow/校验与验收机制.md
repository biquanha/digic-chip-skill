# 校验与验收机制

本章用于约束 skill 的交付质量，避免平台只停留在“有模板”。

## 一、静态校验

静态校验重点确认：
- 目录结构完整
- 引用关系完整
- Makefile / filelist / tb_top 互相一致
- 没有明显残留占位符

## 二、动态校验

如果环境允许，建议继续做动态校验：
- 远端编译是否通过
- 是否成功生成 `simv`
- 是否成功生成 `FSDB`
- 是否存在基本 PASS 判据

## 三、验收最小集

对于新模块第一版平台，建议以下项必须满足：
- smoke case 可运行
- 有至少一种比较机制
- 有基础日志摘要
- 有波形导出能力
- 有后续扩展点说明

## 四、回归前检查

在平台进入日常回归前，应继续检查：
- testcase 命名是否统一
- 是否区分 smoke/basic/reset/error
- coverage 是否已有初始规划
- reference model 与 DUT 配置面是否一致
- RAL 是否有最小占位规划
