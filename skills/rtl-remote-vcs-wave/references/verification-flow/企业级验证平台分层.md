# 企业级验证平台分层

本章描述面向企业级模块验证平台的推荐分层，目标不是只把模块“跑通”，而是让平台可复用、可维护、可交接、可回归、可度量。

## 一、基础分层

建议至少拆成以下层次：
- DUT 层：原始 RTL 与 wrapper
- 接口层：interface、clocking block、protocol assertion
- 事务层：sequence item、约束、字段封装
- agent 层：driver、monitor、sequencer
- env 层：多 agent 装配、virtual sequencer、checker
- model 层：reference model、predictor、adapter
- scoreboard 层：比对、统计、错误分类
- register 层：RAL、adapter、predictor
- sequence/test 层：case 分层与场景管理
- report 层：日志摘要、回归统计、覆盖率结果

## 二、企业级要求

相比最小 demo，企业级平台应额外关注：
- 多人维护时目录职责清晰
- 组件命名统一
- 关键日志可 grep
- testcase 命名稳定
- 回归入口固定
- 覆盖率口径一致
- 配置机制可扩展
- 能支撑版本演进和模块迭代

## 三、推荐能力成熟度

### L1：可跑通
- smoke 可运行
- 有 FSDB
- 有基本 PASS/FAIL

### L2：可比较
- 有 reference model / predictor
- 有 scoreboard 摘要
- 能发现主路径错误

### L3：可回归
- case 分层清晰
- 统一回归入口
- 日志与结果汇总稳定

### L4：可度量
- 有 coverage 规划
- 有回归统计
- 有 issue 分类
- 有里程碑验收口径

### L5：可交接
- 有平台说明
- 有新成员接入说明
- 有常见故障处理
- 有维护边界与更新流程
