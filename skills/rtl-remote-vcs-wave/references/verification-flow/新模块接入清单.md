# 新模块接入清单

## DUT 理解清单

- 模块名是什么
- 时钟/复位是什么
- 输入输出接口有哪些
- 是否是 ready/valid 握手
- 是否有寄存器配置接口
- 是否有多通道
- 是否有仲裁/排序/缓存逻辑
- 是否有 flush / reset / backpressure 行为

## 平台设计清单

- 需要几个 interface
- 需要几个 agent
- 哪些 agent 是 active
- 哪些 agent 是 passive
- 是否需要 RAL
- 是否需要参考模型
- scoreboard 对比维度是什么
- coverage 关注哪些功能点

## 最小可运行标准

- 可以编译通过
- 可以跑通一个 smoke testcase
- 能生成 FSDB
- 能通过日志判断 pass/fail
- monitor 能抓到关键输出

## 进一步完善标准

- 有参考模型
- 有 scoreboard
- 有 coverage
- 有多个 testcase
- 有回归脚本
