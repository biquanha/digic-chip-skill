# 总体验证流程

本章节用于指导 agent 面向一个新的 RTL 模块，如何规划完整的 UVM 验证流程。

## 阶段划分

### 阶段 1：需求与接口理解

目标：
- 看懂 DUT 的功能边界
- 明确输入输出接口
- 区分控制面与数据面
- 提取需要验证的行为点

需要产出：
- DUT 接口表
- 时序/握手规则摘要
- 功能点清单
- 风险点清单

### 阶段 2：验证架构设计

目标：
- 决定验证平台拆分方式
- 判断要几个 agent
- 判断是否需要 RAL
- 判断是否需要参考模型
- 判断 scoreboard 如何比对

需要产出：
- env 拓扑
- agent 列表
- monitor 采样点
- model / scoreboard 关系
- coverage 采样策略

### 阶段 3：最小平台搭建

目标：
- 搭出可编译、可仿真、可导波形的最小平台

至少包含：
- interface
- transaction
- sequencer
- driver
- monitor
- agent
- env
- base_test
- tb_top
- sim/Makefile

### 阶段 4：检查机制落地

目标：
- 建立功能正确性的判断方法

至少考虑：
- monitor 采样是否完整
- model 是否能表达黄金行为
- scoreboard 是否能稳定比较
- 错误日志是否可读

### 阶段 5：场景设计

目标：
- 从 smoke 到 corner case 逐步构建 testcase

建议分层：
- smoke
- basic functional
- protocol/error
- random/stress
- reset/interleave
- configuration switch

### 阶段 6：覆盖率与收敛

目标：
- 对照功能点分析覆盖缺口
- 判断 testcase 是否足够

覆盖关注：
- code coverage
- functional coverage
- cross coverage

### 阶段 7：远端回归

目标：
- 在远端 EDA 节点稳定执行回归
- 保留日志、波形、覆盖率产物

## agent 行动建议

当面对新模块时，优先按以下顺序推进：
1. 先理解 DUT
2. 再决定验证架构
3. 先搭最小平台
4. 再补 checking/model/coverage
5. 最后再做大规模回归
