# 真实项目接入示例

本文总结 `uvm-mcdf` 风格工程接入标准 skill 时应关注的要点。

## 工程识别

当拿到一个真实 UVM 工程时，先看：
- 是否存在 `sim/Makefile`
- 是否存在 `rtl.list` 或 `filelist.f`
- testcase 是否通过 `TESTNAME=...` 选择
- 波形输出目录是否固定
- 日志目录是否固定

## 本次已验证的样例

参考工程：`uvm-mcdf`

远端执行方式：
- 进入 `sim/`
- `make clean`
- `make TESTNAME=case0`

## 成功产物

该类工程跑通后，通常至少会有：
- 编译日志
- 仿真日志
- `simv` 或等价仿真执行文件
- `*.fsdb`

## 验收判据

对于 UVM 工程，不要只看 make 返回码。
至少再检查：
- 日志中有 `TestCase Passed`
- `UVM_ERROR : 0`
- `UVM_FATAL : 0`
- scoreboard error count 为 0
- FSDB 文件存在且大小正常

## skill 接入动作建议

agent 应按以下顺序工作：
1. 识别入口脚本和 filelist
2. 识别 testcase 选择方式
3. 先跑一个最小 case
4. 提取日志中的 pass/fail 规则
5. 再决定是否要套标准模板或仅做适配层
