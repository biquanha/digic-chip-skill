# 平台交接与维护指南

本章面向“这个 skill 后续要交给别人维护”的场景。

## 交接时至少要有的材料

- 平台结构图
- DUT 验证范围说明
- 当前 testcase 分层
- 当前 coverage 状态
- 当前已知限制
- 常见问题与处理方法
- 回归入口命令
- 工具版本与环境依赖

## 维护边界

维护者需要明确：
- 哪些目录可直接扩展
- 哪些模板是标准骨架，不建议随意改
- 哪些脚本是公共入口
- 哪些日志字段是回归系统依赖字段

## 变更管理建议

每次平台升级，建议记录：
- 新增了哪些 case
- 新增了哪些 checker / assertion
- 参考模型是否变化
- 覆盖率口径是否变化
- Makefile / 回归命令是否变化

## 新成员接手时的建议顺序

1. 先读平台总览与 DUT 说明
2. 再读回归入口与日志判据
3. 再看 env / agent / scoreboard / RAL
4. 最后再改 testcase 和约束
