# eda01 工具盘点与扩展建议

本文用于记录 `eda01` 上对企业级模块验证平台有价值的工具，以及在 skill 中的接入建议。

## 一、已明确验证可用的工具

以下工具已在前序流程中实际使用并验证过：
- `Synopsys VCS Q-2020.03-SP2`
- `Synopsys Verdi R-2020.12-SP1`
- `fsdbreport`
- `license` 模块环境

这些工具已足以支撑：
- RTL/UVM 编译与仿真
- FSDB 导出
- 波形摘要分析
- 基础 smoke 验收

## 二、建议继续在 eda01 现场确认的工具类型

如果 `eda01` 环境具备，建议继续纳入 skill 的企业级能力地图：

### 1. 覆盖率与回归
- `urg`
- 作用：覆盖率合并、回归统计
- 建议场景：夜间回归、版本里程碑验收

### 2. 前仿真编译辅助
- `vlogan`
- `vhdlan`
- 作用：分步编译、问题定位更细

### 3. Lint / CDC
- `SpyGlass` / `sg_shell`
- 作用：lint、CDC、RDC
- 建议场景：多时钟、异步复位、复杂控制模块

### 4. 综合与等价性相关
- `dc_shell` / `design_vision`
- `fm_shell`
- 作用：综合可实现性、等价性检查协同
- 建议场景：验证平台与综合网表阶段联动

### 5. 时序相关
- `pt_shell` / `PrimeTime`
- 作用：时序约束与路径认知的辅助闭环
- 建议场景：关键路径模块、时序敏感设计

### 6. 形式验证
- `VC Formal` / `vcf`
- 或 `JasperGold`
- 作用：边界性质、死锁、公平性、协议不变量验证

### 7. Cadence / Siemens 备选仿真器
- `xrun` / `irun`
- `qverilog` / `vsim`
- 作用：多仿真器兼容性或项目环境适配

## 三、skill 中的接入原则

### 已验证工具
直接写入标准 flow：
- `VCS`
- `Verdi`
- `fsdbreport`

### 未现场确认工具
先以“扩展建议”方式纳入 skill：
- 给出用途
- 给出适用场景
- 给出接入时机
- 不在主 flow 中假定一定存在

## 四、建议后续补齐的自动探测

建议未来再补一个远端探测脚本，例如：
- 探测 `command -v`
- 探测 `module avail`
- 输出标准工具摘要表

这样 agent 在接手新机器时，能先自动判断：
- 哪些 flow 可直接用
- 哪些 flow 只能保留为规划项
