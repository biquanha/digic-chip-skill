# 回归与覆盖率收敛

企业级平台不只需要 testcase，还需要长期回归和覆盖率收敛策略。

## 回归分层

建议拆分：
- smoke 回归：提交级别，快速检查主链路
- basic 回归：日常回归
- full 回归：夜间或周回归
- milestone 回归：版本冻结前验收

## 回归集管理

建议固定：
- case list
- seed 策略
- 失败重跑策略
- 日志目录结构
- 结果摘要格式

## 覆盖率收敛思路

### 第 1 阶段：先补空洞
- 先找 0 hit / 低 hit 的功能点
- 判断是 stimulus 不够还是 checker 没接上

### 第 2 阶段：按功能点补 directed case
- 对关键模式、关键错误路径加定向 case

### 第 3 阶段：用随机和约束拉广度
- 调整约束
- 多 seed 跑回归

### 第 4 阶段：评估是否存在无效 bin
- 标注 unreachable
- 与设计确认 spec 边界

## 验收时要同步输出

agent 在做阶段总结时，建议至少输出：
- 当前回归层次
- 当前核心 case 数量
- 当前 coverage 缺口
- 下一轮优先补齐项
