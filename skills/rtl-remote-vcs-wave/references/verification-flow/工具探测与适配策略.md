# 工具探测与适配策略

当 skill 要落到一台新的工作机或 EDA 节点时，第一步不应直接开跑复杂 flow，而应先做工具探测。

## 推荐脚本

```bash
bash skills/rtl-remote-vcs-wave/scripts/probe_eda_tools.sh --remote eda01 --output eda01_probe.txt
```

## 探测目标

至少确认以下几类工具是否存在：
- 仿真：`vcs`、`xrun`、`qverilog`
- 波形：`verdi`
- 覆盖率：`urg`
- 分步编译：`vlogan`、`vhdlan`
- lint/cdc：`spyglass`、`sg_shell`
- 形式：`vcf`、`jaspergold`
- 综合/等价/时序：`dc_shell`、`fm_shell`、`pt_shell`

## 适配原则

### 已探测到
若工具存在，可把对应章节从“规划项”升级为“可执行 flow”。

### 未探测到
若工具不存在：
- 保留文档建议
- 不把它写死到主流程
- 在任务输出中明确“当前机器不可用”

## agent 输出建议

完成探测后，agent 应至少总结：
- 已可直接使用的工具
- 建议继续接入的工具
- 当前机器缺失的关键能力
- 主 flow 是否仍以 `VCS + Verdi + fsdbreport` 为核心
